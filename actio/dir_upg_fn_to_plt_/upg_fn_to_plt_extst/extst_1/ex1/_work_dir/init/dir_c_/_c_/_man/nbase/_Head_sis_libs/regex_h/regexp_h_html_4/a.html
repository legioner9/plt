Теги:  Регулярное выражение

1, регулярные выражения в C / C ++

Ни один из стандартных C и C ++ не поддерживает использование регулярных выражений, но существование регулярных выражений в некоторых сценах может принести много удобства, например, решение имени пользователя и пароля в регистрации пользователя, см. Соответствует. Nameout.

Хотя не поддерживается, некоторые из библиотек могут помочь нам использовать регулярные выражения в C.

Например, Perl-совместимые библиотеки регулярных выражений с Linux.

Функция обработки регулярных выражений в языке C имеет REGCOMP (), Regexec (), regfree () и regerror ().

2, REGCOMP () Функция - составление регулярных выражений

【прототип】:

int regcomp(regex_t *compiled, const char *pattern, int cflags);

  • 2

[Роль]: эта функция компилирует указанную шаблон регулярного выражения в виде определенного формата данных, который делает соответствие более эффективным. Функция Regeexec использует эти данные для сопоставления совмещения режима в целевой текстовой строке. Выполните успешно возвращение 0. Быть

【Описание параметра】:

  • Regex_t - это структурный тип данных, который хранит скомпилированное регулярное выражение, а его элемент RE_NSUB используется для хранения количества субтрагматических выражений в регулярных выражениях.
  • Шаблон указывает на регулярную строку экспрессии, которая будет скомпилирована.
  • CFLAG имеет следующие четыре значения:

REG_EXTENDED:Сопоставьте функцию более мощного расширения регулярного выражения.
REG_ICASE: Игнорируйте случай при сопоставлении письма.
REG_NOSUB: Не нужно хранить соответствующий результат.
REG_NEWLINE: Определите новую строку, поэтому «$» может совпадать с расстояния, «^» может начать с начала строки.

【прототип】:


int regexec (regex_t *compiled, char *string, size_t nmatch, regmatch_t matchptr [], int eflags)

【Описание параметра】:

  • Компиляция - это регулярное выражение, которое функция RegComp скомпилирована.
  • Строка - это целевая строка.
  • Nmatch - это длина структурного массива regmatch_t.
  • MatchPtr Regmatch_T Тип структурного массива, хранить информацию о местоположении с соответствующими текстовыми строками.
  • EFLAGS имеет два значения: REG_NOTBOL, REG_NOTEOL.

4, regfree () Функция - выпуск регулярных выражений

[Функциональный прототип]:

void regfree (regex_t *compiled)
  • 1

[Роль]: При использовании скомпилированного хорошего регулярного выражения или когда вы хотите перекомпилировать другие регулярные выражения, мы можем использовать эту функцию для очистки содержимого конструкции Regex_T, указывающую на компиляцию, если она перекомпилирована, должна сначала очистить Regex_T структура.

5, Regeror () Функция

[Функциональный прототип]:

size_t regerror (int errcode, regex_t *compiled, char *buffer, size_t length)
  • 1

[Роль] При выполнении ошибок генерации RegComp или Regexec эта функция может быть вызвана для возврата строки, содержащей сообщение об ошибке.

【Описание параметра】:

  • ERRCODE - это код ошибки, возвращаемого функцией RegComp и Regexec.
  • Компиляция - это регулярное выражение, которое было составлено с функцией RegComp, которая может быть нулевым.
  • Буфер указывает на пространства памяти для хранения строки сообщений об ошибках.
  • Длина указывает на длину буфера, если длина этого сообщения об ошибке больше, чем это значение, а функция regerror автоматически усекает превышение строки, но он все еще возвращает длину полной строки. Таким образом, мы можем использовать следующий метод, чтобы сначала получить длину неправильной строки.

6, regmatch_t структура

【прототип】:

typedef struct
{
   regoff_t rm_so;
   regoff_t rm_eo;
} regmatch_t;

【Переменные элементов】:

  • Члены rm_so Соответствующие текстовые строки для хранения в начале позиции целевой строки;
  • Конечное положение rm_eo;

[Примечание]: Обычно мы определяем набор таких структур в форме массива. Подкорректирующие выражения также включены в наши регулярные выражения. Устройство Array 0 хранит положение основного регулярного выражения выражения, а блок позади последовательно хранит позицию подстройки.

7, Пример

#include<stdio.h>
#include<sys/types.h>
#include<regex.h>

int my_match(char* pattern,char* buf){
  int status,i;
  int flag=REG_EXTENDED;
  regmatch_t pmatch[1];
  const size_t nmatch=1;
  regex_t  reg;
  // составление обычного режима
  regcomp(&reg,pattern,flag);
  // Выполните регулярные выражения и сравнения кэширования
  status=regexec(&reg,buf,nmatch,pmatch,0);
  / / Распечатать соответствующую строку
  for(i=pmatch[0].rm_so;i<pmatch[0].rm_eo;++i){
    putchar(buf[i]);
  }
  printf("\n");
  regfree(&reg);
  return status;
}

int main(){
  char pattern[1024]="^[1-9][0-9]{10}$";
  char buf[1024]="41509030127";
  int status=my_match(pattern,buf);
  if(status==REG_NOMATCH)
    printf("No match!\n");
  else if(0 == status){
    printf(«Соответствие успеха! \ N»);
  }
  return 0;
}
